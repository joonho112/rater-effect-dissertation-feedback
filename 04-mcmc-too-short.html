4  A single short MCMC chain cannot carry the HRM

The concern in this chapter is narrow: it is about how long the MCMC sampler ran, and what that means for trusting the specific numerical comparisons drawn from it. This discussion appears on pages 60–61.

4.1 What the dissertation does

The HRM is estimated with the R package immer, which — as the dissertation notes — “provides one chain” (p. 60). That single chain runs 1,000 iterations, of which the first 300 are discarded as burn-in (the warm-up phase whose draws are thrown away while the sampler is still finding its way around the posterior). The remaining ~700 draws are the entire posterior sample — the pool from which every estimate, interval, and between-condition comparison is read.

For a sense of scale, the benchmarks cited on the same page: immer’s own default is 500,000 iterations with 200,000 burn-in; Patz et al. (2002) used 10,000 / 1,000; Casabianca et al. used 105,000 / 5,000 (p. 60). The chain is between one-fifteenth and one-hundredth as long as established practice. The priors are diffuse: \(\phi_r \sim N(0, 10^2)\), \(\psi_r \sim N(0.4, 10^2)\), and \(\sigma^2 \sim \text{InverseGamma}(0.001, 1)\) (p. 61).

To check convergence, the dissertation reports a Gelman–Rubin \(\hat{R}\) — a diagnostic that compares between-chain to within-chain variance, with values near 1.0 signaling that chains started from different places have settled on the same distribution. But a real \(\hat{R}\) needs at least two chains, and only one was run; so the reported value is computed “by splitting a single chain into three segments and treat[ing] them as independent chains” (p. 61). Even that weaker substitute flags trouble: “some parameters, such as \(a_i\), exhibited high Rhat values, indicating potential convergence issues” (p. 61). The dissertation concludes the focal results were not strongly affected — but that claim is hard to verify with the evidence at hand.

4.2 Why this threatens the conclusion

The core issue is that 700 draws from one chain is not 700 independent draws. A Markov chain produces autocorrelated samples — each draw resembles the one before it — so it must run long enough for that correlation to decay before it behaves like a random sample. The effective sample size corrects for this: if the integrated autocorrelation time is \(\tau\), then \(\text{ESS} \approx 700/\tau\). For a hierarchical IRT model with hundreds of correlated parameters (an ability for every examinee, an item vector for every item, and every \(\phi_r\) and \(\psi_r\)), estimated by the single-site Metropolis–Hastings sampler immer uses, \(\tau\) is commonly in the tens to hundreds. At \(\tau = 20, 50, 100\), those ~700 retained draws give an ESS of only 35, 14, and 7 — a few dozen genuinely independent draws at best, dwindling to single digits. The remedy is built into that same arithmetic: because \(\text{ESS} \approx N/\tau\), a chain ten times longer would lift these to roughly 350, 140, and 70. The problem is not autocorrelation itself but that 700 draws is too thin a pool to absorb it, and lengthening the chain is exactly what raises the ESS.

That matters because every reported posterior mean carries a Monte-Carlo standard error — the uncertainty from using a finite, autocorrelated sample rather than the true posterior: \(\text{MCSE} = \text{posterior SD} / \sqrt{\text{ESS}}\). With \(\sigma_\phi \approx 0.5\), the MCSE at those ESS values is about 0.085, 0.134, and 0.189 — the same size as the between-condition \(\phi\) differences of roughly 0.05–0.16 interpreted as meaningful. In other words, the Monte-Carlo noise in the estimate is as large as the reported signal. Figure 4.1 shows both halves of this.

suppressPackageStartupMessages({ library(ggplot2); library(patchwork) })
source("R/common.R")

set.seed(2026)
N_KEEP <- 700L; SD_PHI <- 0.5; BURN <- 300L; LEN <- 1000L
DIFF_LO <- 0.05; DIFF_HI <- 0.16
ess  <- function(tau) N_KEEP / tau
mcse <- function(tau) SD_PHI / sqrt(ess(tau))
tau_seq <- seq(1, 150, length.out = 400)

ar1 <- function(rho, n, sd = SD_PHI, mu = 0) {          # single-site-MH-like autocorrelation
  x <- numeric(n); x[1] <- rnorm(1, mu, sd); inn <- rnorm(n, 0, sd * sqrt(1 - rho^2))
  for (t in 2:n) x[t] <- mu + rho * (x[t - 1] - mu) + inn[t]; x
}
rho_typ <- (50 - 1) / (50 + 1)                          # τ = 50
chA <- ar1(rho_typ, LEN); chB <- ar1(rho_typ, LEN)
mA <- mean(chA[(BURN + 1):LEN]); mB <- mean(chB[(BURN + 1):LEN])

# Panel A — two single chains disagree
dfA <- rbind(data.frame(it = 1:LEN, phi = chA, chain = "Chain A"),
             data.frame(it = 1:LEN, phi = chB, chain = "Chain B"))
pA <- ggplot(dfA, aes(it, phi, colour = chain)) +
  annotate("rect", xmin = 0, xmax = BURN, ymin = -Inf, ymax = Inf, fill = "grey85", alpha = .6) +
  annotate("text", x = BURN/2, y = 1.7, label = "burn-in\n(300 discarded)", size = 2.7,

           fill = "white", alpha = .85, label.size = 0, lineheight = .9,
           label = sprintf("the two post-burn-in means differ by %.2f\n— a single chain cannot reveal this", abs(mA - mB))) +

pB <- ggplot(dfB, aes(tau, mcse)) +

A note on the figure: it is an illustrative surrogate calibrated to the regime of single-site Metropolis–Hastings on a hierarchical posterior — not a re-run of the immer model, since the data are not available. The ESS and MCSE relationships are exact algebra; the chain-disagreement panel illustrates what is possible in this regime, not a claim about the sampler’s exact \(\tau\). The arithmetic, worked from first principles, reaches the same place.

Two further points. First, the high \(\hat{R}\) for \(a_i\) cannot be safely bracketed: in a hierarchical model \(a_i\), \(\theta\), \(\xi\), \(\phi\), and \(\psi\) are coupled in the joint posterior, so a sampling problem in one is a warning about the whole. The dissertation does not leave this unexamined: it re-checks with longer chains and finds the focal patterns stable (p. 60), and it compares the \(a_i\)-estimated against the \(a_i\)-fixed specification across 20 replications, where the results were consistent (a moderate, positive correlation, with the mean \(\phi\) and \(\psi\) holding up; p. 61). What those checks cannot do is substitute for a convergence statistic: a stable pattern across longer runs, or matching means across replications, is a robustness observation — it tells a reader the result is reproducible — not a formal \(\hat{R}\)/ESS demonstration that the coupled sampler has converged. Second, the diffuse priors compound the issue: \(\text{InverseGamma}(0.001, 1)\) on \(\sigma^2\) is extremely diffuse — its mode sits near 1 and, with shape \(0.001 < 1\), its mean is not even finite, so it spreads enormous prior mass over large \(\sigma^2\) (a very heavy right tail) rather than piling mass near zero. The Limitations section (p. 74) flags that immer’s default inverse-gamma prior “has raised concerns in previous research,” citing Gelman (2006) by name. The one nuance to add is which form Gelman criticized: the pathology is the \(\text{InverseGamma}(\epsilon, \epsilon)\) case — both parameters tiny — whose pile-up of mass near zero is the genuinely long-criticized behavior, and that is not the \(\text{InverseGamma}(0.001, 1)\) used here. With a scale of 1, this prior behaves the opposite way near the boundary, so Gelman’s specific warning does not land on it. Even so, such a heavy-tailed prior on \(\sigma^2\) can slow mixing, and when convergence is uncertain its influence on \(\phi\) and \(\psi\) cannot be cleanly separated from sampling noise.

NoteThe strongest counterargument

The record shows convergence was not ignored. The high \(\hat{R}\) for some parameters is reported openly in the Limitations (p. 74); additional checks with longer chains found the focal patterns stable (p. 60); the \(a_i\)-estimated specification was compared against the \(a_i\)-fixed specification across 20 replications (p. 61) with consistent results; and the dissertation flags that immer’s default inverse-gamma prior “has raised concerns in previous research,” citing Gelman (2006). This is more convergence diligence than most applied MFRM papers show.

The issue was never awareness; it is the form of the evidence. “The longer chains looked stable” and “the means matched across 20 replications” are robustness observations, not convergence statistics — they tell a reader the result is plausible, not that it is settled. The journal version needs those same checks formalized: a true multi-chain \(\hat{R}\), bulk- and tail-ESS, and an MCSE for each focal \(\phi\) and \(\psi\), so that “the patterns were stable” becomes “ESS = X, MCSE = Y, and each interpreted difference exceeds its MCSE by a comfortable margin.” The judgment-laden part is done; what remains is to report it in the currency the reviewer counts in.

4.3 What would fix it

TipFix-it
  • Run ≥ 4 independent chains from dispersed starting values — the prerequisite for a meaningful \(\hat{R}\) — each much longer than 1,000/300 (treat Patz’s 10,000/1,000 or Casabianca’s 105,000/5,000 as a floor, and extend if diagnostics warrant). The obstacle is that immer provides only one chain by default, so there are two realistic routes to the four. The immediate, low-cost one: run immer ≥ 4 times from different seeds and dispersed starting values, then pool the runs as chains and compute the diagnostics across them. The more thorough one — and the better long-run investment — is to re-implement the HRM in a native multi-chain sampler (Stan, JAGS, or NIMBLE), which runs the chains jointly and yields proper multi-chain \(\hat{R}\), bulk-/tail-ESS, and MCSE directly.
  • Report, for every focal parameter (\(\phi_r\), \(\psi_r\)), the true multi-chain \(\hat{R}\), the bulk- and tail-ESS, and the MCSE — and state in the text that each interpreted difference exceeds its MCSE by a comfortable margin.
  • Show trace plots and per-chain posterior-density overlays for the focal parameters as supplementary material — the most direct evidence a reader can inspect.
  • Run a prior-sensitivity analysis for \(\phi\) and \(\psi\); replace \(\text{InverseGamma}(0.001,1)\) with a half-Normal or half-Cauchy on the SD scale (which mixes far better), and consider a \(\log(\psi)\) / positivity-constrained parameterization to avoid boundary pathologies.
  • Add posterior-predictive checks — simulate replicated ratings from the posterior and compare to the observed data — as independent evidence of fit and reach.

4.4 What a reviewer will ask

WarningWhat a reviewer will ask

“What are the effective sample size and Monte-Carlo standard error of \(\phi\) and \(\psi\) at 700 draws from a single chain — and are the between-condition differences you interpret larger than that Monte-Carlo error? And how was an \(\hat{R}\) computed from a single chain?”

immer provides a single chain by default, and a short run is an understandable choice under real time and compute limits — a software-and-resources constraint, not a methodological error. The point is only what the journal version needs to show. Under the current setup the ESS is likely single digits to low tens, the MCSE is of the same order as the reported differences, and the \(\hat{R}\) came from a split-chain approximation that itself flagged problems. The goal of the revision is to answer with specific numbers — ESS = X, MCSE = Y, interpreted difference = Z, and a large Z/Y. This is achievable, and getting there makes the HRM results persuasive rather than merely plausible.

tail-ESS, and an MCSE for each focal $\phi$ and $\psi$, so that "the patterns were stable"
becomes "ESS = X, MCSE = Y, and each interpreted difference exceeds its MCSE by a comfortable
margin." The judgment-laden part is done; what remains is to report it in the
currency the reviewer counts in.
:::

## What would fix it

::: {.callout-tip}
## Fix-it
- **Run ≥ 4 independent chains** from dispersed starting values — the prerequisite for a
  meaningful $\hat{R}$ — each much longer than 1,000/300 (treat Patz's 10,000/1,000 or
  Casabianca's 105,000/5,000 as a floor, and extend if diagnostics warrant). The obstacle
  is that `immer` provides only one chain by default, so there are two realistic routes to
  the four. The immediate, low-cost one: run `immer` ≥ 4 times from different seeds and
  dispersed starting values, then pool the runs as chains and compute the diagnostics
  across them. The more thorough one — and the better long-run investment — is to
  re-implement the HRM in a native multi-chain sampler (Stan, JAGS, or NIMBLE), which runs
  the chains jointly and yields proper multi-chain $\hat{R}$, bulk-/tail-ESS, and MCSE
  directly.
- **Report, for every focal parameter** ($\phi_r$, $\psi_r$), the true multi-chain
  $\hat{R}$, the bulk- and tail-ESS, and the MCSE — and state in the text that each
  interpreted difference exceeds its MCSE by a comfortable margin.
- **Show trace plots and per-chain posterior-density overlays** for the focal parameters
  as supplementary material — the most direct evidence a reader can inspect.
- **Run a prior-sensitivity analysis** for $\phi$ and $\psi$; replace
  $\text{InverseGamma}(0.001,1)$ with a half-Normal or half-Cauchy on the SD scale (which
  mixes far better), and consider a $\log(\psi)$ / positivity-constrained parameterization
  to avoid boundary pathologies.
- **Add posterior-predictive checks** — simulate replicated ratings from the posterior
  and compare to the observed data — as independent evidence of fit and reach.
:::

## What a reviewer will ask

::: {.callout-warning}
## What a reviewer will ask
"What are the effective sample size and Monte-Carlo standard error of $\phi$ and $\psi$ at
700 draws from a single chain — and are the between-condition differences you interpret
larger than that Monte-Carlo error? And how was an $\hat{R}$ computed from a single chain?"
:::

`immer` provides a single chain by default,
and a short run is an understandable choice under real time and compute limits — a
software-and-resources constraint, not a methodological error. The point is only what the
*journal* version needs to show. Under the current setup the ESS
is likely single digits to low tens, the MCSE is of the same order as the reported differences,
and the $\hat{R}$ came from a split-chain approximation that itself flagged problems.
The goal of the revision is
to answer with specific numbers — ESS = X, MCSE = Y, interpreted difference =
Z, and a large Z/Y. This is achievable, and getting there makes the
HRM results persuasive rather than merely plausible.
